# **易経診断システム：マスター・データ仕様書**

## **1\. 情報構造の設計思想**

本システムは、ユーザーの問いを「単なる占い」としてではなく、「メタ認知による状況認識と意思決定の支援」として構造化する。そのため、バックエンドでは易経の本質である4つの観点を持ち、フロントエンドではユーザーに馴染みのある4つのタグ（ボタン）として提示する。

### **バックエンド・カテゴリーの定義**

1. **時流・天命 (Time & Mandate)**:  
   * 外部環境の時間的推移、不可避な運命、バリアントの空間における現在の座標。  
2. **徳・処世 (Virtue & Conduct)**:  
   * 社会的文脈における倫理的応答、戦略的行動、アドラー心理学的な「課題の分離」。  
3. **観照・修養 (Contemplation & Cultivation)**:  
   * 内面的な意識の変容、右脳的アプローチ、重要性を下げて「観客」として生きる知恵。  
4. **場・根底 (Field & Foundation)**:  
   * 物理的環境、身体性、潜在意識の基盤、エネルギー（周波数）の状態。

## **2\. JSONフィールド・定義表**

各卦のJSON（hexagram\_nn.json）に含まれるべき要素。

| フィールド | 内容 | UIへの反映（PDF参照） |
| :---- | :---- | :---- |
| id / gua\_name | 卦番号 / 日本語名 | 診断タイトル |
| kanji / reading | 漢字 / 読み | 卦名（ルビ付き） |
| english\_title | 英語の卦名 | サブタイトル |
| binary\_code | 陰陽のバイナリ(例: 111111\) | 卦象（八卦画）の生成用 |
| trigrams | 上卦・下卦の属性データ | 卦の成り立ち・属性テーブル |
| oracle\_text | 卦辞（漢文および書き下し） | 冒頭の経典引用 |
| core\_description | 卦の全体的な概要 | 解釈セクションの導入 |
| diagnoses | 4つのカテゴリー別診断 | ジャンル選択ボタンによる出し分け |
| lines | 6つの爻（コウ）の解釈 | 個別の爻（変爻）の診断結果 |
| inspiration | キーワードとシンボル | 右脳的インスピレーション枠 |
